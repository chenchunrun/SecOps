# 两高一弱检测详情

## 高危漏洞检测

### 检测范围
CVSS ≥ 7.0 的漏洞

### 检测方法
```
MCP 调用: cyberspace-search
查询: hostname="*.example.com" && vuln!=""
字段: vuln (漏洞信息)
```

### 高危漏洞类型

| 类型 | CVE 示例 | 风险等级 |
|------|----------|----------|
| 远程代码执行 (RCE) | Log4Shell, Spring4Shell | 🔴 严重 |
| SQL 注入 | - | 🔴 严重 |
| 未授权访问 | Redis/MongoDB 未授权 | 🔴 高 |
| 任意文件读取 | - | 🔴 高 |
| SSRF | - | 🟡 中高 |

### 漏洞资产关联查询

```
# 查询某 CVE 影响的资产
cyberspace-search: vuln="CVE-2024-XXXX"

# 查询某资产的所有漏洞
cyberspace-search: hostname="target.com" && vuln!=""
```

## 高危端口检测

### 高危端口定义

| 端口 | 服务 | 风险描述 | 检测建议 |
|------|------|----------|----------|
| 22 | SSH | 远程登录，暴力破解风险 | 限制来源 IP |
| 23 | Telnet | 明文传输，严禁使用 | 必须关闭 |
| 445 | SMB | 永恒之蓝等漏洞 | 内网隔离 |
| 1433 | MSSQL | 数据库暴露 | 禁止公网 |
| 3306 | MySQL | 数据库暴露 | 禁止公网 |
| 3389 | RDP | Windows 远程，漏洞频发 | 限制访问 |
| 5432 | PostgreSQL | 数据库暴露 | 禁止公网 |
| 6379 | Redis | 未授权访问风险 | 设置密码 |
| 27017 | MongoDB | 未授权访问风险 | 设置认证 |
| 9200 | Elasticsearch | 数据泄露风险 | 禁止公网 |

### 批量端口扫描

```
MCP 调用: ops_portscan
参数:
  target: "<资产IP>"
  ports: [22, 23, 445, 1433, 3306, 3389, 5432, 6379, 27017, 9200]
  timeout_ms: 2000
```

## 弱口令检测

> ⚠️ **注意**: 弱口令检测需要授权，仅限自有资产

### 检测服务

| 服务 | 默认端口 | 常见弱口令 |
|------|----------|------------|
| SSH | 22 | root:root, admin:admin |
| RDP | 3389 | Administrator:123456 |
| MySQL | 3306 | root:root, root:空 |
| Redis | 6379 | 无密码 |
| FTP | 21 | anonymous:空 |
| Tomcat | 8080 | admin:admin, tomcat:tomcat |

### 检测策略

1. 先通过端口扫描确定开放服务
2. 针对开放服务检测默认凭证
3. 输出弱口令清单和加固建议

```markdown
**检测范围**: XX 个资产

| 漏洞编号 | 影响资产 | CVSS | 状态 |
|----------|----------|------|------|

### 2. 高危端口统计
| 端口 | 服务 | 暴露数量 | 风险 |
|------|------|----------|------|

### 3. 弱口令统计
| 服务 | 资产 | 账号 | 状态 |
|------|------|------|------|
| SSH | 1.2.3.4 | root | 弱口令 |

### 4. 修复建议
1. **高优先级**: 立即修复 RCE 漏洞
2. **高优先级**: 关闭不必要的高危端口
3. **中优先级**: 修改所有弱口令
```
