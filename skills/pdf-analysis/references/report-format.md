# PDF 分析报告格式规范

## 报告结构（9个章节）

| # | 章节 | 必含 | 内容要点 |
|---|------|------|----------|
| 1 | 标题 + 威胁评级 | ✅ | 等级 + 威胁类型 + 风险评分 |
| 2 | 文件基本信息 | ✅ | 文件名/大小/哈希/版本/对象统计 |
| 3 | 风险评分明细 | ✅ | 各检测项及分值 |
| 4 | 威胁检测结果 | ✅ | JavaScript/嵌入文件/URL/表单 |
| 5 | 详细分析 | 条件 | JS代码/嵌入内容/钓鱼特征/二维码 |
| 6 | IOC 提取 | ✅ | URL/IP/域名/文件哈希 |
| 7 | MITRE ATT&CK 映射 | ✅ | 技术ID + 名称 + 证据 |
| 8 | 分析存在的问题 | ✅ | 局限性说明 |
| 9 | 结论 | ✅ | 特征总结 + 处置建议 |

## 风险评分标准

| 发现 | 分值 | 说明 |
|------|------|------|
| JavaScript 代码 | +25 | 包含 JS 脚本 |
| 自动执行 JS | +30 | OpenAction 触发 JS |
| 嵌入可执行文件 | +40 | EXE/DLL/BAT 等 |
| 恶意 URL | +20 | 外部链接到可疑域名 |
| 二维码钓鱼 | +25 | 图片中包含钓鱼二维码 |
| 已知 CVE 特征 | +40 | Adobe Reader 漏洞 |
| 加密内容 | +15 | 可能隐藏恶意代码 |
| 表单提交 | +15 | 可能窃取数据 |
| Launch 动作 | +35 | 启动外部程序 |
| 外部流对象 | +20 | 引用外部资源 |

| 分数 | 等级 | 标记 |
|------|------|------|
| ≥80 | Critical | 🔴 |

## 威胁类型分类

| 类型 | 特征 | 主要风险 |
|------|------|----------|
| **漏洞利用型** | JS + 特定对象结构 | 远程代码执行 |
| **钓鱼诱导型** | URL + 二维码 + 伪装文本 | 凭据窃取 |
| **恶意载荷型** | 嵌入可执行文件 | 恶意软件感染 |
| **信息窃取型** | 表单 + 外部提交 | 数据泄露 |

## 8. 结论

### 特征总结

| 特征 | 描述 |
|------|------|
| 威胁类型 | 嵌入可执行文件 + JavaScript 触发 |
| 攻击目标 | 在用户系统上执行恶意程序 |
| 攻击阶段 | 投递 → 执行 → 下载载荷 |
| 载荷类型 | PE32 可执行文件 |

### 处置建议

1. 🚫 **立即隔离** - 将 PDF 移至隔离区
2. 🔍 **提取分析** - 使用 `/binary-reverse-engineering` 分析嵌入的 EXE
3. 📝 **IOC 共享** - 将 URL 和哈希上报威胁情报平台
4. 🔒 **策略加固** - 禁用 PDF 阅读器的 JavaScript
5. 📢 **安全提醒** - 提醒用户不要打开可疑 PDF
```
