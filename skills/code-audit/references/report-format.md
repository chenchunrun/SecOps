## 3. 漏洞详情

### 3.1 [严重] {漏洞标题}

- **文件**: `path/to/file.py:123`
- **CWE**: CWE-89 SQL注入
- **CVSS**: 9.8

**问题代码**:
```python
# 第 123-125 行
query = f"SELECT * FROM users WHERE id = {user_id}"
cursor.execute(query)
```

**漏洞描述**:
用户输入直接拼接到 SQL 查询中，攻击者可注入恶意 SQL 语句。

**攻击示例**:
```
user_id = "1 OR 1=1; DROP TABLE users;--"
```

**修复建议**:
```python
# 使用参数化查询
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))
```

**参考链接**:
- https://cwe.mitre.org/data/definitions/89.html

## 5. 安全加固建议

### 5.1 输入验证
- 所有用户输入必须验证和清理
- 使用白名单验证而非黑名单

### 5.2 数据库安全
- 始终使用参数化查询
- 最小权限原则配置数据库账户

### 5.3 认证与会话
- 实施多因素认证
- 使用安全的会话管理

### 5.4 依赖管理
- 定期更新依赖版本
- 使用依赖漏洞扫描工具

## 风险等级定义

| 等级 | CVSS | 典型漏洞 | 处理时限 |
|------|------|---------|---------|
| 严重 | 9.0-10.0 | RCE、SQL注入、认证绕过 | 立即 |
| 高危 | 7.0-8.9 | XSS、SSRF、敏感信息泄露 | 24小时 |
| 中危 | 4.0-6.9 | CSRF、路径遍历、弱加密 | 1周 |
| 低危 | 0.1-3.9 | 信息泄露、配置问题 | 下版本 |

## 必需元素

每个漏洞必须包含：
- [ ] 文件位置（路径:行号）
- [ ] 风险等级
- [ ] CWE 编号
- [ ] 问题代码片段
- [ ] 漏洞描述
- [ ] 修复建议（含代码示例）

## 格式要求

1. 使用 Markdown 格式
2. 代码块标注语言类型
3. 表格对齐整洁
4. 按风险等级排序漏洞
