# IP 威胁评分详细规则

> 评分体系采用 0-100 分制，与业界标准对齐。参考 [Microsoft Defender TI](https://learn.microsoft.com/en-us/defender/threat-intelligence/reputation-scoring)、[IPQualityScore](https://www.ipqualityscore.com/ip-fraud-score-risk-checker) 等行业实践。

## 评分原则

1. **历史行为具有预测价值**：曾用于恶意活动的基础设施可能被重复使用
2. **时间衰减**：历史情报随时间推移权重降低
3. **多源验证**：单一来源可疑，多源确认提升置信度
4. **当前状态优先**：实时扫描结果优先于历史情报

## 基础指标

| 指标 | 分值 | 说明 |
|------|------|------|
| 多源标记恶意 (≥3 源) | +40 | 被多个情报源标记 |
| 多源标记恶意 (2 源) | +30 | 被 2 个情报源标记 |
| 单源标记恶意 | +20 | 仅 1 个情报源标记 |
| C2 标签 | +30 | 命令控制服务器 |
| 恶意样本关联 > 10 | +25 | 大量样本通信 |
| 恶意样本关联 1-10 | +15 | 少量样本通信 |
| 高风险国家/地区 | +10 | 根据地理位置 |
| 防弹主机 ASN | +20 | 已知恶意托管 |
| 近期活跃 (7天内) | +10 | 最近有恶意活动 |
| 开放高危端口 | +10 | 如 445/3389 等 |

## 匿名器/代理指标

> 参考 [MaxMind IP Anonymizer Risk Data](https://support.maxmind.com/hc/en-us/articles/4408261981339-IP-Anonymizer-Risk-Data)、[IPinfo Proxy Detection](https://ipinfo.io/data/proxy-vpn-detection)

| 类型 | 分值 | 说明 |
|------|------|------|
| Tor 出口节点 | +15 | 高度匿名，常被滥用 |
| 公共代理 (Public Proxy) | +15 | 开放代理，高风险 |
| 商业 VPN | +10 | 匿名但有合法用途 |
| 数据中心/托管 IP | +5 | 非住宅 IP，需关注 |
| 住宅代理 | +10 | 可能是僵尸网络节点 |

**组合评分**：
- VPN + 恶意情报 = 基础分 + 匿名分（叠加）
- 仅 VPN 无恶意情报 = 仅计匿名分（不叠加恶意分）

## 黑客工具检测指标

> **评分原则**: 仅端口开放 = 线索（低分），需继续验证

| 指标 | 分值 | 验证程度 | 说明 |
|------|------|----------|------|
| C2 指纹确认 (JARM+证书+响应) | +40 | 已确认 | 多重指纹匹配 |
| C2 指纹部分匹配 | +25 | 高度可疑 | JARM 或证书匹配其一 |
| C2 端口开放 + 异常响应 | +15 | 可疑 | 有可疑响应 |
| **仅 C2 端口开放** | +5 | 待验证 | 需进一步验证 |
| RAT 指纹确认 | +35 | 已确认 | Banner/协议特征匹配 |
| RAT 端口开放 + 异常响应 | +15 | 可疑 | 有可疑响应 |
| **仅 RAT 端口开放** | +5 | 待验证 | 需进一步验证 |
| 挖矿协议确认 | +25 | 已确认 | Stratum 协议匹配 |
| **仅挖矿端口开放** | +5 | 待验证 | 需进一步验证 |
| 多个黑客工具端口 (≥3) | +10 | 可疑 | 多个可疑端口组合 |

## 历史情报时间衰减

当威胁情报为历史记录（端口当前关闭）时，应用时间衰减因子：

| 情报时间 | 衰减系数 | 说明 |
|----------|---------|------|
| 7 天内 | 1.0 | 无衰减，视为活跃 |
| 8-30 天 | 0.8 | 轻微衰减 |
| 31-90 天 | 0.5 | 中度衰减 |
| 91-180 天 | 0.3 | 显著衰减 |
| 180 天以上 | 0.1 | 仅作历史参考 |

**计算方式**：
```
最终分数 = 基础分 × 衰减系数
```

**示例**：
- C2 标签 (+30) + 情报 60 天前 → 30 × 0.5 = +15
- 多源恶意 (+40) + 情报 10 天前 → 40 × 0.8 = +32

## 情报与扫描结果矛盾处理

| 场景 | 情报结果 | 扫描结果 | 评分处理 |
|------|---------|---------|---------|
| A | 端口有 C2 活动 | 端口开放 | 全额计分 |
| B | 端口有 C2 活动 | 端口关闭 | 应用时间衰减 |
| C | 无恶意情报 | 端口开放 | 仅计"端口开放" (+5) |
| D | 无恶意情报 | 端口关闭 | 0 分 |

## 风险等级判定

| 分数 | 等级 | 颜色 | 建议 |
|------|------|------|------|
| 0-20 | 低风险 | 🟢 | 持续监控 |
| 21-40 | 中风险 | 🟡 | 深入调查 |
| 41-60 | 高风险 | 🟠 | 建议阻断 |
| 61-100 | 严重 | 🔴 | 立即阻断 |

## PTR 记录分析

| PTR 特征 | 说明 | 风险提示 |
|----------|------|----------|
| 无 PTR 记录 | 未配置反向解析 | 可疑，正规服务通常有 PTR |
| 通用 PTR | `xxx.static.provider.com` | VPS/云主机，需关注 |
| 动态 PTR | `dyn-xxx.isp.com` | 家用宽带，可能是肉鸡 |
| 自定义 PTR | `mail.company.com` | 检查是否与正向解析匹配 |
| 可疑 PTR | 随机字符串 | 高度可疑 |

> ⚠️ PTR 由 IP 所有者设置，可被伪造，仅作参考

## 高风险 ASN 特征

| 特征 | 风险等级 | 说明 |
|------|---------|------|
| 防弹主机商 | 高 | 常被恶意软件使用 |
| 免费 VPS 提供商 | 中 | 常被滥用 |
| Tor 出口节点 ASN | 中 | 匿名网络 |
| 已知恶意托管商 | 高 | 黑名单 ASN |

---

## 防弹托管识别

| 托管域名数 | 判断 | 说明 |
|-----------|------|------|
| < 10 | 正常 | 普通托管 |
| 10-100 | 可疑 | 共享主机或 CDN |
| > 100 | 高度可疑 | 可能是防弹托管 |
| > 1000 | 防弹托管 | Fast-Flux 或恶意基础设施 |
