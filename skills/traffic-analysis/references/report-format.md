# 网络流量分析报告格式规范

## 报告结构（8个章节）

| # | 章节 | 必含 | 内容要点 |
|---|------|------|----------|
| 1 | 标题 + 威胁评级 | ✅ | 分析目标 + 风险等级 |
| 2 | 流量概览 | ✅ | 时间范围/协议分布/会话统计 |
| 3 | 异常发现 | ✅ | C2模式/外泄/扫描等 |
| 4 | 时间线 | 条件 | 事件响应时需要 |
| 5 | IOC 提取 | ✅ | IP/域名/URL/JA3/文件哈希 |
| 6 | MITRE ATT&CK 映射 | ✅ | 技术ID + 名称 |
| 7 | 分析存在的问题 | ✅ | 局限性说明 |
| 8 | 结论 | ✅ | 特征总结 + 处置建议 |

---

## 评级阈值

| 等级 | 标记 | 条件 |
|------|------|------|
| Critical | 🔴 | 确认 C2 通信 / 数据外泄 / 主动攻击 |
| High | 🟠 | 高度可疑 C2 / 横向移动 |
| Medium | 🟡 | 异常模式需进一步调查 |
| Low | 🟢 | 未发现明显威胁 |

---

```markdown
**分析时间**: YYYY-MM-DD HH:MM

## 2. 异常发现

### C2 通信特征

| 指标 | 发现 | 详情 |
|------|------|------|
| 长连接 | ✅ | IP X.X.X.X 连接 Y 分钟 |
| Beaconing | ✅ | 每 60s 连接一次 |
| 非标准端口 | ✅ | HTTP over 8443 |

### 数据外泄特征

| 指标 | 发现 | 详情 |
|------|------|------|
| 大量上传 | ✅ | 上传 X MB 到 Y.Y.Y.Y |
| DNS 隧道 | ❌ | - |
```

```
## 6. 分析存在的问题

- [ ] 部分加密流量无法解密

## 7. 结论

### 特征总结

| 特征 | 描述 |
|------|------|
| 攻击类型 | C2 通信 + 数据外泄 |
| C2 协议 | HTTPS (端口 443) |
| Beaconing 周期 | 60 秒 |

### 处置建议

1. 🚫 **立即阻断** - 防火墙封锁 IOC 中的 IP/域名
2. 🔍 **内部排查** - 检查受影响主机
3. 📝 **情报共享** - 将 IOC 上报威胁情报平台
```
