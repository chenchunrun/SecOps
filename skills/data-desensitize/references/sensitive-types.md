# 敏感数据类型定义

## 网络标识类

### PUBLIC_IP - 公网 IP 地址
- **正则**: `\b(?!10\.|172\.(1[6-9]|2[0-9]|3[01])\.|192\.168\.)[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\b`
- **示例**: `8.8.8.8`, `1.2.3.4`, `203.0.113.50`
- **排除**: 内网 IP、环回地址 (127.x.x.x)、广播地址

### PRIVATE_IP - 内网 IP 地址
- **范围**:
  - `10.0.0.0/8` (10.x.x.x)
  - `172.16.0.0/12` (172.16.x.x - 172.31.x.x)
  - `192.168.0.0/16` (192.168.x.x)
- **示例**: `192.168.1.100`, `10.0.0.1`, `172.16.0.50`

### DOMAIN - 域名
- **正则**: `\b[a-zA-Z0-9][-a-zA-Z0-9]*(\.[a-zA-Z0-9][-a-zA-Z0-9]*)+\b`
- **示例**: `example.com`, `api.internal.corp.cn`, `mail.company.org`
- **排除**:
  - 公共域名可选保留：`google.com`, `github.com`
  - 文件扩展名：`.md`, `.json`

### URL - 完整 URL
- **处理方式**: 分解后分别脱敏域名、路径参数中的敏感信息
- **示例**: `https://api.company.com/user/12345` → `https://[DOMAIN_1]/user/[USERID_1]`

## 个人身份类

### EMAIL - 电子邮箱
- **正则**: `\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b`
- **示例**: `admin@company.com`, `user.name@corp.cn`

### PHONE - 手机号码
- **中国大陆**: `1[3-9]\d{9}`
- **带区号**: `+86-1[3-9]\d{9}`, `86-1[3-9]\d{9}`
- **示例**: `13800138000`, `+86-13912345678`

### IDCARD - 身份证号
- **18位**: `[1-9]\d{5}(19|20)\d{2}(0[1-9]|1[0-2])(0[1-9]|[12]\d|3[01])\d{3}[\dXx]`
- **示例**: `110101199001011234`, `320106198505152345`

### PERSON - 人名
- **中文姓名**: 2-4 个汉字，常见姓氏开头
- **英文姓名**: 首字母大写的单词组合
- **示例**: `张三`, `李明`, `John Smith`
- **注意**: 需结合上下文判断，避免误判普通词汇

## 金融类

### BANKCARD - 银行卡号
- **长度**: 16-19 位数字
- **正则**: `\b[3-6]\d{15,18}\b`
- **示例**: `6222021234567890123`, `4111111111111111`

### CREDENTIAL - 凭据/密码
- **模式匹配**:
  - `password=xxx`, `passwd: xxx`
  - `secret=xxx`, `token=xxx`
  - `key=xxx` (非公钥)
- **示例**: `password=MyP@ssw0rd!`, `db_password: secret123`
- **弱口令列表** (可选保留):
  ```
  123456, password, admin, root, 12345678,
  qwerty, abc123, 111111, 123123, admin123
  ```

### APIKEY - API 密钥
- **模式**:
  - `sk-[a-zA-Z0-9]{32,}` (OpenAI)
  - `AKIA[A-Z0-9]{16}` (AWS)
  - `ghp_[a-zA-Z0-9]{36}` (GitHub)
  - `xox[baprs]-[a-zA-Z0-9-]+` (Slack)
- **通用**: 长度 > 20 的随机字符串，出现在 key/token/secret 上下文中

## 组织类

### ORG - 组织/公司名称
- **模式**:
  - 中文：`xx公司`, `xx集团`, `xx银行`, `xx大学`
  - 英文：`XXX Inc.`, `XXX Corp.`, `XXX Ltd.`
- **示例**: `某某科技有限公司`, `Acme Corporation`

### PRODUCT - 产品/系统名称
- **上下文判断**: 出现在"系统"、"平台"、"产品"附近的专有名词
- **示例**: `XX 安全平台`, `YY 管理系统`

## 时间地点类

### ADDRESS - 地址
- **中国地址**: 省/市/区/街道/门牌号
- **示例**: `北京市海淀区中关村大街1号`
- **注意**: 通常只脱敏详细地址，保留省市级别

### GPS - 经纬度坐标
- **正则**: `[-+]?\d{1,3}\.\d{4,},\s*[-+]?\d{1,3}\.\d{4,}`
- **示例**: `39.9042, 116.4074`

## 不脱敏的内容

以下内容通常**不需要脱敏**：

1. **版本号**: `v1.0.0`, `2.3.4`
2. **端口号**: `:8080`, `:443`
3. **时间戳**: `2024-01-01 12:00:00`
4. **HTTP 状态码**: `200`, `404`, `500`
5. **公共服务域名**: `google.com`, `github.com` (可配置)
6. **技术术语**: `localhost`, `127.0.0.1` (环回地址)
7. **文档元数据**: 文件大小、行数等

## 上下文判断示例

| 原文 | 判断 | 理由 |
|------|------|------|
| `version: 1.2.3.4` | 不脱敏 | 版本号格式 |
| `connect to 1.2.3.4` | 脱敏 | IP 地址上下文 |
| `file.json` | 不脱敏 | 文件扩展名 |
| `api.company.json` | 脱敏 | 域名（虽然以 .json 结尾但是子域名） |
| `port 3306` | 不脱敏 | 端口号 |
| `id: 3306` | 看上下文 | 可能是用户 ID |
