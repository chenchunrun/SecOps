# URL 威胁分析报告格式规范

## 快速分析 vs 深度分析报告

| 报告类型 | 适用场景 | 章节数 |
|---------|---------|-------|
| 快速分析报告 | 日常排查、初次研判 | 5 章节 |
| 深度分析报告 | 明确要求全面评估、需要视觉证据或扩线 | 8 章节 |

**主要区别**：
- 快速分析只保留最小必要章节：基础信息、核心发现、结论建议、IOC、局限性
- 深度分析在快速分析基础上追加：截图、归因增强、扩线、深度局限性
- `domain-analysis` 仍是域名年龄与 WHOIS 的权威来源，URL 报告中必须明确其来源属性

**快速分析强约束**：
- 默认仅执行快速分析；只有用户明确要求深度分析，或快速分析明确给出升级建议信号时，才允许进入深度分析阶段
- 快速分析仅允许执行 5 类动作：URL 解析与静态特征分析、条件式内容获取与跳转分析、页面内容检测与规避技术检测、`domain-analysis`、URL 威胁情报查询
- 完成上述步骤后，必须立即输出快速分析报告并结束
- 快速分析中禁止继续调用页面截图、WebSearch 归因增强、DNS 历史查询、威胁扩线、同 IP / 同证书 / 相似标题等扩展侦察

---

## 联用结果来源标注规则（必须遵守）

当 `url-analysis` 联用 `domain-analysis`、`ip-analysis` 或其他下游技能时，最终报告必须保证结果可追溯。

### 最小标注要求

- 明确 **主分析技能**，例如：`url-analysis`
- 明确 **联用技能**，例如：`domain-analysis`、`ip-analysis`
- 对关键补充结论标注来源 skill，不得让读者误以为这些结果是 `url-analysis` 默认直接产出

### 关键结论来源约束

- 域名年龄 / WHOIS / 注册商 / 域名侧 DNS 结果，标注为 `domain-analysis` 来源
- IP 地理位置 / ASN / IP 威胁情报结果，标注为 `ip-analysis` 来源
- URL 解析 / 跳转分析 / 页面钓鱼检测 / 规避技术检测 / URL 风险评分，默认视为 `url-analysis` 主分析结果

### 推荐写法

- 在“URL 基础信息”或“结论与处置建议”章节中增加一行：`**联用技能**: domain-analysis（域名侧权威补充）, ip-analysis（关联 IP 快速补充）`
- 在补充结论旁直接标注来源，例如：`（来源：domain-analysis）`、`（来源：ip-analysis）`
- 如果某项结果来自联用技能，仅可写成“补充结果”或“权威来源结果”，不要写成 `url-analysis` 默认步骤

### 禁止写法

- 只给出 WHOIS / ASN / IP 情报结果，不标来源
- 把联用 skill 的结果平铺进 URL 主分析结论，导致主结论与补充结论混淆
- 把 `domain-analysis` 或 `ip-analysis` 的结果表述成 `url-analysis` 默认直接产出

**处置建议**:
1. [+] 持续监控 / 核实业务用途

## 检查状态写法规范（必须遵守）

| 状态 | 适用场景 | 写法示例 |
|------|----------|----------|
| 已检测（未发现风险） | 检查已执行且结果正常 | `已检测（未发现风险）` |
| 已检测（发现风险） | 检查已执行且发现问题 | `已检测（发现风险） - 品牌仿冒` |
| 不适用 | 当前 URL 场景不需要该项 | `不适用（IP 直连 URL）` |
| 未执行 | 因模式约束或数据缺失未执行 | `未执行（快速分析未包含截图）` |

**禁止**：
- 不要把“已执行且无风险”写成“未检测”
- 不要把“未执行”伪装成阴性结论

---

## IOC 填写约束

- `domain-analysis` 结果中的域名年龄 / WHOIS 必须标注为域名侧权威来源
- `ip-analysis` 结果中的 IP 地理位置 / ASN / IP 威胁情报必须标注为 IP 侧补充来源
- 只有实际拿到最终跳转目标时，才填写“最终跳转目标”
- 只有实际拿到 IP / 哈希 / 关联文件时，才写入 IOC
- 如果情报源只返回数量而无明细，明确写“未返回具体明细”

---

## Markdown 格式规范（必须遵守）

**段落分隔**：
- 不同主题之间必须空一行
- 不要把多个信息点挤在同一行

**键值对数据**：
- 单对象信息优先用列表
- 每个列表项完整内容保持在同一行

**表格使用场景**：
- 适合多条记录对比：检测项、扩线结果、评分明细
- 不适合单个 URL 的基础字段

**截图写法**：
- 截图章节必须使用 Markdown 图片语法
- 禁止只用普通文本描述截图链接

**动态文本转义**：
- Markdown 表格中的 `|` 必须替换为全角 `｜`
- 表格中的换行必须替换为空格
