# IP 威胁分析报告格式规范

## 快速分析 vs 深度分析报告

| 报告类型 | 适用场景 | 章节数 |
|---------|---------|-------|
| **快速分析报告** | 日常排查、初次研判 | 4 章节 |
| **深度分析报告** | 全面评估、入侵分析 | 8 章节（增加端口详情） |

**主要区别**：
- 快速分析仅保留：基础信息、威胁情报、结论建议、分析局限性
- 深度分析有完整端口扫描结果和风险评分明细
- 深度分析的风险评估包含时间衰减和端口因素

**快速分析强约束**：
- 默认仅执行快速分析；只有用户明确要求“深度分析”或明确要求“端口扫描”时，才允许进入深度分析阶段
- 快速分析仅允许执行 3 类操作：IP 验证、地理位置/ASN 查询、威胁情报查询
- 完成上述 3 步后，必须立即输出快速分析报告并结束
- 快速分析中禁止继续调用主动端口扫描、C2/RAT 端口探测、DNS 历史查询、同 IP 域名分析、C 段分析、关联域名递归分析、关联 URL 递归分析，以及任何额外的 cyberspace-search 扩展侦察

---

## 联用结果来源标注规则（必须遵守）

当 `ip-analysis` 联用 `domain-analysis`、`url-analysis` 或其他下游技能时，最终报告必须保证结果可追溯。

### 最小标注要求

- 明确 **主分析技能**，例如：`ip-analysis`
- 明确 **联用技能**，例如：`domain-analysis`、`url-analysis`
- 对关键补充结论标注来源 skill，不得让读者误以为这些结果是 `ip-analysis` 默认直接产出

### 关键结论来源约束

- IP 地理位置 / ASN / IP 威胁情报 / 端口扫描结果，默认视为 `ip-analysis` 主分析结果
- 域名 WHOIS / 域名年龄 / 注册商 / 域名侧 DNS 结果，标注为 `domain-analysis` 来源
- URL 跳转分析 / 页面内容检测 / 规避技术检测 / URL 威胁情报，标注为 `url-analysis` 来源

### 推荐写法

- 在“基础信息”或“结论与处置建议”章节中增加一行：`**联用技能**: domain-analysis（域名侧补充）, url-analysis（URL 侧补充）`
- 在补充结论旁直接标注来源，例如：`（来源：domain-analysis）`、`（来源：url-analysis）`
- 如果某项结果来自联用技能，仅可写成“补充结果”或“联用结果”，不要写成 `ip-analysis` 默认步骤

### 禁止写法

- 只给出域名 WHOIS / 域名年龄 / URL 页面结论，不标来源
- 把联用 skill 的结果平铺进 IP 主分析结论，导致主结论与补充结论混淆
- 把 `domain-analysis` 或 `url-analysis` 的结果表述成 `ip-analysis` 默认直接产出

# 快速分析报告模板（5 章节）

```markdown
# [!] IP 威胁分析报告

**威胁等级**: [HIGH] (评分: 55)
**分析时间**: YYYY-MM-DD HH:MM
**分析模式**: 快速分析
---
## 1. 基础信息
- **IP 地址**: <IP>
- **IP 类型**: 公网地址
- **地理位置**: <国家> / <省/州> / <城市>
- **ASN**: AS<号码> (<组织名>)
- **PTR 记录**: <PTR> 或 无
---
## 2. 威胁情报
| 来源 | 判定 | 标签 | 首次发现 | 最后活跃 |
|------|------|------|----------|----------|
| <来源名> | malicious | c2, scanner | YYYY-MM-DD | YYYY-MM-DD |

**关联情报数量**: X 条
**关联家族/标签**: <family/tag list> 或 未返回
**关联 C2 端口**: 端口号 (协议)
**情报时效**: 最新活动 YYYY-MM-DD (<状态>)
---
## 3. 结论与处置建议
**威胁类型**: 扫描器 / C2 服务器 / 僵尸网络
**风险等级**: 中 / 高

**处置建议**:
1. [+] 持续监控
2. [*] 建议深度分析确认端口状态
3. [!] 如确认恶意，阻断该 IP
---
## 4. IOC 汇总
**IP 地址**: <IP>
**PTR 记录**: <PTR> 或 无
**关联域名**: <list> 或 未发现
**关联端口**: <ports> 或 未发现
**关联 URL**: <list> 或 未发现
---
## 5. 分析局限性
未进行深度扫描，缺少如下信息：
- 未进行主动端口扫描
- 未查询 DNS 解析历史
- 未进行 C 段分析
```

# 深度分析报告模板（8 章节）

**关联 IOC 填写约束**：
- 只有拿到具体哈希（MD5/SHA1/SHA256）时，才写入“关联样本哈希”
- 家族名（如 Gh0st、Zegost、ShellLoader）必须写入“关联家族名”
- 标签（如 c2、cc、botnet、scanner）必须写入“关联标签”
- 如果情报源只返回数量、未返回明细，明确写“未返回具体明细”，不要臆造内容
- `domain-analysis` 结果中的域名 WHOIS / 域名年龄 / 注册商必须标注为域名侧补充来源
- `url-analysis` 结果中的 URL 跳转 / 页面内容 / URL 威胁情报必须标注为 URL 侧补充来源

```markdown
# [!] IP 威胁分析报告

**威胁等级**: [HIGH] (评分: 55)
**分析时间**: YYYY-MM-DD HH:MM
**分析模式**: 深度分析
---
## 1. 基础信息
- **IP 地址**: <IP>
- **IP 类型**: 公网地址
- **地理位置**: <国家> / <省/州> / <城市>
- **ASN**: AS<号码> (<组织名>)
- **ISP**: <ISP名称>
- **PTR 记录**: <PTR> 或 无
- **匿名性**: VPN / Tor / 代理 / 无
- **主分析技能**: ip-analysis
- **联用技能**: domain-analysis（域名侧补充）/ url-analysis（URL 侧补充）/ 无
---
## 2. 威胁情报
| 来源 | 判定 | 标签 | 首次发现 | 最后活跃 |
|------|------|------|----------|----------|
| <来源名> | malicious | c2, scanner | YYYY-MM-DD | YYYY-MM-DD |

**关联情报数量**: X 条
**关联家族/标签**: <family/tag list> 或 未返回
**关联 C2 端口**: 端口号 (协议)
**情报时效**: 最新活动 YYYY-MM-DD (<状态>)

**联用补充结果**:
- **域名 WHOIS / 域名年龄 / 注册商**: <summary> 或 未联用（来源：domain-analysis）
- **URL 跳转 / 页面内容 / URL 威胁情报**: <summary> 或 未联用（来源：url-analysis）
---
## 3. 开放端口
| 端口 | 服务 | 状态 | Banner |
|------|------|------|--------|
| 22/tcp | SSH | 开放 | OpenSSH 8.2 |
| 80/tcp | HTTP | 开放 | nginx 1.18 |
| 443/tcp | HTTPS | 关闭 | - |

**高危端口检测**: 4444 (C2指纹) - 开放
---
## 4. 关联 IOC
**关联域名** (<X> 个):
- domain[.]example[.]com

**关联样本数量**: X 个

**关联样本哈希**:
- <sha256/md5 list> 或 未返回具体样本哈希

**关联家族名**:
- Gh0st
- Zegost

**关联标签**:
- c2
- cc

**关联 URL** (<X> 个):
- hxxp://domain[.]com/payload
---
## 5. 风险评估
| 指标 | 结果 | 基础分 | 衰减 | 最终分 |
|------|------|--------|------|--------|
| 多源标记恶意 | ✅ X源 | +40 | ×1.0 | +40 |
| C2 标签 | ✅ | +30 | ×1.0 | +30 |
| 恶意样本关联 | 2 个 | +15 | - | +15 |
| 开放高危端口 | ✅ | +10 | - | +10 |
| **总分** | | | | **95** |

时间衰减：7天内 ×1.0 / 8-30天 ×0.8 / 31-90天 ×0.5 / 91-180天 ×0.3 / 180天+ ×0.1
---
## 6. 情报与扫描结果对比
**场景**: 情报显示端口 4444 有 C2 活动，扫描确认端口开放

**判定**: 高风险 - 情报与扫描一致，C2 活跃
---
## 7. 结论与处置建议
**威胁类型**: C2 服务器
**活跃状态**: 活跃
**风险等级**: 高

**结论来源说明**:
- IP 主结论来自 `ip-analysis`
- 域名侧补充结果来自 `domain-analysis`（如已联用）
- URL 侧补充结果来自 `url-analysis`（如已联用）

**处置建议**:
1. [x] 立即阻断 - 在防火墙中封锁该 IP
2. [x] 内部排查 - 检查是否有主机与该 IP 通信
3. [x] 取证分析 - 保留通信日志和流量样本
4. [x] 持续监控 - 加入威胁情报监控列表
---
## 8. 分析局限性
由 LLM 根据实际分析情况动态填写，例如：
- 情报源返回数据不完整
- 部分端口未扫描
- DNS 历史记录缺失等
- 若未联用 `domain-analysis`，则域名侧权威结果视为未补充
- 若未联用 `url-analysis`，则 URL 侧补充结果视为未补充
```

## Markdown 格式规范（必须遵守）

**段落分隔**：
- 不同主题/段落之间必须用**空行**分隔（双换行）
- 禁止把多个信息项挤在同一行
- 每个独立信息点单独成段

**键值对数据格式**（如基础信息）：
- 使用列表格式，不要用表格
- 每个列表项的完整内容必须在同一行，禁止在列表项内部换行
- 列表项之间不要空行

**表格使用场景**：
- ✅ 适合：多条记录对比（威胁情报多源、端口列表、评分明细）
- ❌ 不适合：单个对象的键值对数据

**禁止使用的格式**：
- ❌ 禁止使用 `>` 引用块语法
- ❌ 禁止滥用代码块包裹普通文本
- ❌ 禁止在列表项内部换行

## 风险评分标准

| 指标 | 分值 |
|------|------|
| 多源标记恶意 (≥3 源) | +40 |
| 多源标记恶意 (2 源) | +30 |
| 单源标记恶意 | +20 |
| C2 标签 | +30 |
| 恶意样本关联 > 10 | +25 |
| 恶意样本关联 1-10 | +15 |
| 高风险国家/地区 | +10 |
| 防弹主机 ASN | +20 |
| 商业 VPN | +10 |
| Tor/公共代理 | +15 |
| 开放高危端口 | +10 |

## 评级阈值

| 分数 | 等级 | 标记 |
|------|------|------|
| 61+ | Critical | [!] |
| 41-60 | High | [!] |
| 21-40 | Medium | [*] |
| 0-20 | Low | [+] |
