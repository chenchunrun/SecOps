# PDF 报告格式规范

## 报告结构

专业的 PDF 报告应包含以下结构：

| 序号 | 章节 | 必需 | 说明 |
|------|------|------|------|
| 1 | 封面页 | 是 | 标题、副标题、作者、日期、机构 |
| 2 | 执行摘要 | 是 | 关键发现和结论概述 |
| 3 | 正文章节 | 是 | 详细分析内容（可多个章节） |
| 4 | 表格/图表 | 条件 | 数据可视化展示 |
| 5 | 结论与建议 | 是 | 总结发现并提出行动建议 |
| 6 | 附录 | 可选 | 补充技术细节 |

---

## 封面页规范

封面页应包含以下元素：

- **标题**: 简洁明了，概括报告主题（建议 ≤20 字）
- **副标题**: 补充说明，可包含目标对象或时间范围
- **作者/团队**: 报告撰写者或团队名称
- **日期**: 报告生成日期
- **机构**: 所属组织名称（可选）

**示例**:
```
标题: IP 威胁分析报告
副标题: 针对 45.33.32.156 的深度分析
作者: 安全分析团队
日期: 2024年1月15日
机构: 安全运营中心
```

---

## 标题层级规范

| 级别 | 用途 | 字号 | 示例 |
|------|------|------|------|
| 一级标题 | 主要章节 | 22pt | 一、执行摘要 |
| 二级标题 | 章节分段 | 18pt | 1.1 分析背景 |
| 三级标题 | 细分内容 | 16pt | 1.1.1 事件概述 |
| 四级标题 | 详细条目 | 14pt | 1.1.1.1 具体说明 |

**命名规范**:
- 使用中文数字或阿拉伯数字编号
- 标题简洁，避免过长
- 保持层级结构清晰

## 正文排版规范

### 段落格式

- **首行缩进**: 24 磅（约 2 个汉字宽度）
- **行距**: 1.5 倍行距
- **对齐**: 两端对齐
- **字号**: 12pt

### 列表格式

无首行缩进的内容适用于：
- 列表项
- 键值对信息
- 代码或命令

```python
# 带首行缩进（默认）
manager.add_content("这是正常段落，会有首行缩进。")

# 无首行缩进
manager.add_content("列表项内容", first_line_indent=False)
```

## 表格规范

### 表格样式

| 元素 | 样式 |
|------|------|
| 表头背景 | 浅灰色 (#F5F5F5) |
| 数据行 | 交替行背景色 |
| 边框 | 1pt 黑色实线 |
| 对齐 | 居中对齐 |

### 表格说明

- 每个表格应有说明文字
- 自动编号格式: `Table <N> 说明文字`
- 说明位于表格下方

### 最佳实践

1. 表头行应清晰标识列含义
2. 数据单元格内容简洁
3. 列宽自动适应，避免内容截断
4. 控制表格宽度，不超出页面边距

---

## 图片规范

### 图片尺寸

| 场景 | 建议宽度 | 说明 |
|------|----------|------|
| 全宽图片 | 14cm | 默认尺寸，适合大多数场景 |
| 中等图片 | 10cm | 适合截图、示意图 |
| 小型图片 | 6cm | 适合图标、小型图表 |

### 图片说明

- 每个图片应有说明文字
- 自动编号格式: `Figure <N> 说明文字`
- 说明位于图片下方

### 图片格式支持

- PNG: 推荐，支持透明背景
- JPEG: 适合照片类图片
- GIF: 支持，但不推荐用于报告

## 安全报告特定规范

### 威胁等级标记

| 等级 | 颜色标记 | 分数范围 |
|------|----------|----------|
| Critical | 红色 | 61+ |
| High | 橙色 | 41-60 |
| Medium | 黄色 | 21-40 |
| Low | 绿色 | 0-20 |

### IOC 格式化

- URL 使用防点击格式: `hxxp://example[.]com`
- IP 地址使用方括号: `192.168[.]1.1`
- 哈希值完整展示，不截断

### 处置建议格式

建议使用编号列表：
1. 立即行动项
2. 短期措施
3. 长期改进

---

## 文档信息

### 页眉

- 位置: 页面顶部右侧
- 内容: 文档标题或报告名称
- 字号: 10.5pt
- 分隔线: 页眉下方细灰色线

### 页脚

- 位置: 页面底部居中
- 内容: 页码
- 格式: `Page N`
- 起始: 从正文第一页开始计数（跳过封面）

## 输出检查清单

生成 PDF 报告前，检查以下项目：

- [ ] 封面信息完整（标题、作者、日期）
- [ ] 章节结构清晰
- [ ] 表格和图片有说明文字
- [ ] 页眉文本已设置
- [ ] 文件名有意义且包含日期
- [ ] 敏感信息已脱敏处理
