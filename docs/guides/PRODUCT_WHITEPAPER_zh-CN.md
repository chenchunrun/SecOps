# SecOps Agent 产品白皮书

**版本：** 2026-07-14  
**产品形态：** 终端原生的安全运营与工程协作智能体  
**项目地址：** <https://github.com/chenchunrun/SecOps>

## 摘要

SecOps Agent 是基于 Crush 终端智能体基础构建的独立 SecOps 扩展项目。产品将大模型辅助、工程工具调用与安全运营治理结合在同一终端工作流中，使安全分析、运维协作和代码任务能够在受控权限、风险评估和审计追踪下完成。

它并不将智能体视为无边界的命令执行器，而是将每一次敏感操作纳入能力授权、风险判定、执行策略和审计记录的链路。该设计面向需要兼顾效率、可追溯性和安全边界的安全运营团队、平台工程团队和研发组织。

## 1. 产品定位

SecOps Agent 是一个终端优先的协作运行时，覆盖以下工作模式：

- **OPS：** 面向运维、环境诊断、监控与基础设施查询。
- **SEC：** 面向安全分析、事件响应、合规检查与风险研判。
- **CODE：** 面向工程实现、代码理解与开发协作。
- **AUTO：** 根据任务上下文在上述能力之间路由。

产品的目标不是替代现有 SIEM、云平台或研发工具，而是在这些工具之间提供一个具备策略约束的智能协作层。

## 2. 核心能力

### 2.1 SecOps 工具集

系统集成 18 类 SecOps 工具能力，覆盖安全扫描、监控、日志分析、证书审计、密钥审计、合规检查、事件支持、MITRE ATT&CK 推理、事件评估、部署与基础设施查询等典型任务。

工具实现以独立 Go 模块组织，并由统一的能力、权限和审计机制接入运行时，避免安全工具成为绕过治理边界的特殊入口。

### 2.2 安全技能与专业智能体

产品提供 35 项安全技能：

- 28 项防御型技能，涵盖事件响应、威胁情报、资产与攻击面、恶意软件检测、代码与供应链安全、脱敏和报告生成。
- 7 项红队技能，用于已授权的侦察、漏洞验证和社会工程规划场景。

红队类能力必须经过显式授权：智能体会声明意图、要求确认目标范围和交战规则，并记录相应审计事件。未满足授权、超出范围或缺少回滚计划的请求应被拒绝。

### 2.3 风险感知的执行控制

SecOps Agent 将工具调用和命令执行置于风险感知控制之下：

- 基于能力模型判断调用者是否拥有目标操作所需权限。
- 使用多因素风险评估支持敏感操作的许可、确认或拒绝决策。
- 对高风险路径、远程目标和危险操作施加额外限制。
- 将策略与审计中间件复用于本地和远程执行路径，减少实现漂移。

该模型的核心价值是将“模型建议执行”转化为“经策略评估后执行”。

### 2.4 可治理的执行后端

系统支持本地、Docker 和 SSH 三类执行后端：

- **本地执行：** 适用于受控工作站上的开发、分析和诊断任务。
- **Docker 执行：** 为隔离型任务提供容器化执行边界。
- **SSH 执行：** 支持远程运维与检查，并对远程主机和端口实施允许列表等校验。

不同执行后端共享策略验证和审计中间件，使运行上下文、控制要求和结果记录保持一致。

### 2.5 审计、SIEM 与合规

每项受治理操作均可形成审计事件，以支持复核、追责和合规取证。审计能力包括：

- 持久化审计事件模型与存储。
- 合规报告生成及 PDF 导出。
- 带敏感信息脱敏的 SIEM 导出。
- ELK、Splunk、Azure Sentinel 和通用 JSON 接收端导出。
- HTTPS 传输要求和认证负载保护等导出端安全约束。

这些能力可将终端中的智能体活动纳入现有安全运营体系，而非形成不可观测的自动化孤岛。

## 3. 技术架构

SecOps Agent 使用 Go 构建，主要组件如下：

| 层级 | 主要职责 |
| --- | --- |
| CLI 与 TUI | 提供终端交互、模式切换和任务路由。 |
| 智能体协调层 | 协调通用智能体、OpsAgent、SecurityExpertAgent 及工具调用。 |
| 工具与能力层 | 提供 SecOps 工具目录、能力注册和权限检查。 |
| 策略与安全层 | 负责风险评估、能力授予和敏感执行约束。 |
| 执行层 | 提供本地、Docker、SSH 后端与共享治理中间件。 |
| 审计与集成层 | 提供 SQLite 持久化、合规报告和 SIEM 导出。 |

系统同时保留多模型提供商、会话工作流、LSP 和 MCP 扩展能力，使其能融入现有研发与安全工具链。

## 4. 典型应用场景

- **安全事件响应：** 汇总日志、分析可疑指标、生成处置建议并保留完整操作审计。
- **云与基础设施巡检：** 查询部署状态、资源与成本信息，在权限策略约束下执行运维操作。
- **代码与供应链审查：** 配合代码审计和 SCA 能力识别风险，并将结论形成可交付报告。
- **合规证据整理：** 输出审计记录、合规报告和可供 SIEM 消费的结构化事件。
- **受控的红队辅助：** 在显式授权和范围约束下，辅助进行攻击面研究和验证计划制定。

## 5. 安全设计原则

- **最小权限：** 能力授予不默认扩散，敏感能力需要显式配置。
- **显式授权：** 红队类活动不依赖隐式意图判断，必须获得用户确认。
- **纵深防御：** 权限、风险、执行策略、远程目标校验和审计共同构成控制链。
- **默认可审计：** 受治理操作能够生成结构化审计信息，并支持外部安全平台导出。
- **安全传输：** SIEM 导出端要求真实 HTTPS 端点，并对数据进行脱敏处理。

## 7. 部署与集成

产品可从源代码构建，并通过 GitHub Releases 分发发布产物。运行时可使用 OpenAI、Anthropic、Google Gemini、Azure OpenAI、Amazon Bedrock 等兼容提供商；也可通过 MCP 接入外部工具和服务。

部署前应根据组织策略配置：

- 模型提供商凭据与网络出口。
- `crush.json` 中的权限、能力授予和 MCP 服务。
- Docker 与 SSH 执行后端的隔离、目标允许列表和凭据管理。
- 审计存储、SIEM 接收端和保留策略。

## 8. 结论

SecOps Agent 将终端智能体的效率与安全运营所需的治理能力结合起来。其差异化价值不在于单独提供某一个扫描或分析功能，而在于通过风险感知权限、可治理执行、多模式专业智能体和审计/SIEM 闭环，使智能自动化能够更可靠地进入安全与工程生产流程。
