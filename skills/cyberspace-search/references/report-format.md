# 网络空间资产搜索报告格式

## 报告结构

```
═══════════════════════════════════════════════════════
网络空间资产搜索报告
═══════════════════════════════════════════════════════

【搜索概要】
查询语句:     <原始查询>
搜索时间:     <时间戳>
结果总数:     <N> 条
返回数量:     <M> 条

【资产统计】
按国家/地区分布:
  - 中国:      XX 个 (XX%)
  - 美国:      XX 个 (XX%)
  - ...

按端口分布:
  - 80/tcp:    XX 个
  - 443/tcp:   XX 个
  - ...

按组件分布:
  - nginx:     XX 个
  - apache:    XX 个
  - ...

【风险资产】
🔴 高风险: X 个
🟡 中风险: X 个
🟢 低风险: X 个

【详细资产列表】
┌──────────────────┬───────┬──────────┬─────────────┬──────────┐
│ IP 地址          │ 端口  │ 组件     │ 国家/地区   │ 风险等级 │
├──────────────────┼───────┼──────────┼─────────────┼──────────┤
│ 1.2.3.4          │ 80    │ nginx    │ CN/Beijing  │ 🟢 低    │
│ 5.6.7.8          │ 3306  │ mysql    │ US/CA       │ 🔴 高    │
│ ...              │ ...   │ ...      │ ...         │ ...      │
└──────────────────┴───────┴──────────┴─────────────┴──────────┘

【风险发现】
1. 敏感端口暴露
   - X 个资产暴露数据库端口 (3306/5432/27017)
   - X 个资产暴露远程管理端口 (22/3389)

2. 老旧版本组件
   - X 个资产运行老旧 Apache (< 2.4)
   - X 个资产运行老旧 PHP (< 7.4)

3. 潜在漏洞资产
   - X 个资产可能受 CVE-XXXX-XXXX 影响

【处置建议】
1. 立即处理 (高风险)
   - 关闭或限制对外暴露的数据库端口
   - 升级存在已知漏洞的组件

2. 计划处理 (中风险)
   - 审查不必要的服务暴露
   - 更新老旧版本软件

3. 持续监控 (低风险)
   - 定期资产扫描
   - 建立资产变更告警

【IOC 汇总】
高风险 IP 列表:
  - 1.2.3.4
  - 5.6.7.8

关联域名:
  - example.com
  - test.example.org

═══════════════════════════════════════════════════════
```

## 风险评级标准

### 高风险 (🔴)

| 条件 | 说明 |
|------|------|
| 数据库端口暴露 | 3306/5432/27017/6379/9200 等 |
| 无认证服务 | Redis/MongoDB 无密码 |
| 已知漏洞版本 | 存在高危 CVE |
| 敏感信息泄露 | Banner 含敏感配置 |
| 工控协议暴露 | Modbus/S7comm 等 |

### 中风险 (🟡)

| 条件 | 说明 |
|------|------|
| 远程管理端口 | 22/23/3389/5900 公网暴露 |
| 老旧软件版本 | 停止维护的版本 |
| 后台管理入口 | admin/管理页面暴露 |
| 调试接口暴露 | phpinfo/debug 页面 |

### 低风险 (🟢)

| 条件 | 说明 |
|------|------|
| 标准 Web 服务 | 80/443 正常服务 |
| 最新版本软件 | 保持更新的组件 |
| 正常配置 | 无明显安全问题 |

## 简洁报告模式

当作为子任务被其他技能调用时，使用简洁模式：

```
【资产搜索摘要】
查询: app:nginx country:CN
结果: 共 50 个资产
  - 高风险: 5 个（数据库暴露）
  - 中风险: 10 个（老旧版本）
  - 低风险: 35 个

高风险资产: 1.2.3.4:3306, 5.6.7.8:27017
```

## 报告场景适配

### 攻击面测绘报告

增加：
- 子域名发现统计
- 服务类型分布
- 潜在入口点分析

### 漏洞资产发现报告

增加：
- CVE 映射
- 影响版本范围
- 修复建议优先级

### 合规审计报告

增加：
- 敏感端口合规性检查
- 数据暴露风险评估
- 整改建议时间线
