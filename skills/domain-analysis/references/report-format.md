# 域名威胁分析报告格式规范

## 快速分析 vs 深度分析报告

| 报告类型 | 适用场景 | 章节数 |
|---------|---------|-------|
| **快速分析报告** | 日常排查、初次研判 | 4 章节 |
| **深度分析报告** | 全面评估、入侵分析 | 8 章节（增加检测详情） |

**主要区别**：
- 快速分析仅保留：域名基本信息、威胁情报、结论建议、分析局限性
- 深度分析有完整检测结果（DGA、同形字、CDN、ICP、子域名）
- 深度分析包含关联分析和 DNS 安全配置
- 快速分析和深度分析都必须输出评分明细与总分

**快速分析强约束**：
- 默认仅执行快速分析；只有用户明确要求“深度分析”时，才允许进入深度分析阶段
- 快速分析仅允许执行 3 类操作：域名格式验证、WHOIS 查询、威胁情报查询
- 完成上述 3 步后，必须立即输出快速分析报告并结束
- 快速分析中禁止继续调用 ICP 查询、DNS 历史、子域名发现、同 IP 域名分析、CDN 检测、DGA 检测、同形字检测、解析 IP 的地理归属扩展分析，以及任何额外的 cyberspace-search 扩展侦察

## 联用结果来源标注规则（必须遵守）

当 `domain-analysis` 联用 `ip-analysis`、`url-analysis` 或其他下游技能时，最终报告必须保证结果可追溯。

### 最小标注要求

- 明确 **主分析技能**，例如：`domain-analysis`
- 明确 **联用技能**，例如：`ip-analysis`、`url-analysis`
- 对关键补充结论标注来源 skill，不得让读者误以为这些结果是 `domain-analysis` 默认直接产出

### 关键结论来源约束

- 域名 WHOIS / 域名年龄 / 注册商 / ICP / DGA / 同形字 / CDN / 子域名发现，默认视为 `domain-analysis` 主分析结果
- IP 地理位置 / ASN / IP 威胁情报 / IP 端口扫描结果，标注为 `ip-analysis` 来源
- URL 跳转分析 / 页面内容检测 / 规避技术检测 / URL 威胁情报，标注为 `url-analysis` 来源

### 推荐写法

- 在“域名基本信息”或“结论与处置建议”章节中增加一行：`**联用技能**: ip-analysis（IP 侧补充）, url-analysis（URL 侧补充）`
- 在补充结论旁直接标注来源，例如：`（来源：ip-analysis）`、`（来源：url-analysis）`
- 如果某项结果来自联用技能，仅可写成“补充结果”或“联用结果”，不要写成 `domain-analysis` 默认步骤

### 禁止写法

- 只给出 IP 地理位置 / ASN / URL 页面结论，不标来源
- 把联用 skill 的结果平铺进域名主分析结论，导致主结论与补充结论混淆
- 把 `ip-analysis` 或 `url-analysis` 的结果表述成 `domain-analysis` 默认直接产出

## 快速分析报告模板（5 章节）

```markdown
# [!] 域名威胁分析报告

**威胁等级**: [HIGH] (评分: 45)
**分析时间**: YYYY-MM-DD HH:MM
**分析模式**: 快速分析
---
## 1. 域名基本信息
- **域名**: <domain>
- **TLD**: .<tld>
- **注册商**: <registrar>
- **注册日期**: YYYY-MM-DD
- **域名年龄**: X 天
- **注册人信息**: <info> 或 隐私保护
---
## 2. 威胁情报
| 来源 | 判定 | 标签 |
|------|------|------|
| <来源名> | malicious | c2, phishing |

**检测率**: X/X 引擎标记恶意
**情报时效**: 最新活动 YYYY-MM-DD (<状态>)
---
## 3. 结论与处置建议
**威胁类型**: 钓鱼域名 / C2 域名 / 恶意域名
**风险等级**: 高

**评分明细**:
- **域名年龄**: +20
- **情报源命中**: +20
- **标签命中（phishing/c2/malware 等）**: +5
- **总分**: 45

**处置建议**:
1. [+] 持续监控
2. [*] 建议深度分析进一步确认
3. [!] 如确认恶意，阻断该域名解析
---
## 4. IOC 汇总
**域名**: <domain>
**解析 IP**: <ip> 或 未获取
**关联 URL**: <list> 或 未发现
---
## 5. 分析局限性
未进行深度扫描，缺少如下信息：
- 未进行 DGA 检测
- 未进行同形字检测
- 未进行 CDN 检测
- 未查询子域名历史
```

## 深度分析报告模板（8 章节）

**报告输出约束**：
- “解析 IP”仅填写本次实际拿到的解析结果；未获取到时明确写“未获取”
- “关联域名”仅填写同 IP 域名、子域名发现或情报源明确返回的关联域名；不要把当前分析域名重复写入“关联域名”
- “关联 URL”仅在情报源或关联分析返回具体 URL 时填写；未返回具体 URL 时写“未发现”
- “ICP 备案”仅针对注册域名（主域名）填写，不对子域名单独臆造备案信息
- 如果某项检查未执行、执行失败或结果缺失，必须明确写“未执行”“查询失败”或“未返回”，不要默认写成阴性结论
- `ip-analysis` 结果中的 IP 地理位置 / ASN / IP 威胁情报 / 端口扫描必须标注为 IP 侧补充来源
- `url-analysis` 结果中的 URL 跳转 / 页面内容 / URL 威胁情报必须标注为 URL 侧补充来源

```markdown
# [!] 域名威胁分析报告

**威胁等级**: [HIGH] (评分: 45)
**分析时间**: YYYY-MM-DD HH:MM
**分析模式**: 深度分析
---
## 1. 域名基本信息
- **域名**: <domain>
- **TLD**: .<tld>
- **注册商**: <registrar>
- **注册日期**: YYYY-MM-DD
- **域名年龄**: X 天
- **注册人信息**: <info> 或 隐私保护
- **ICP 备案**: 有/无
- **主分析技能**: domain-analysis
- **联用技能**: ip-analysis（IP 侧补充）/ url-analysis（URL 侧补充）/ 无
---
## 2. 威胁情报
| 来源 | 判定 | 标签 | 首次发现 | 最后活跃 |
|------|------|------|----------|----------|
| <来源名> | malicious | c2, phishing | YYYY-MM-DD | YYYY-MM-DD |

**检测率**: X/X 引擎标记恶意
**情报时效**: 最新活动 YYYY-MM-DD (<状态>)

**联用补充结果**:
- **IP 地理位置 / ASN / IP 威胁情报 / 端口状态**: <summary> 或 未联用（来源：ip-analysis）
- **URL 跳转 / 页面内容 / URL 威胁情报**: <summary> 或 未联用（来源：url-analysis）
---
## 3. 特殊检测
**DGA 检测**: 未发现 DGA 特征 / 疑似 DGA 域名
**同形字检测**: 未发现同形字攻击 / 疑似仿冒 "<品牌>"
**动态 DNS 检测**: 未发现动态 DNS 托管特征 / 命中高风险动态 DNS 服务 (<服务商>) / 命中中风险动态 DNS 服务 (<服务商>)
**CDN 检测**: 是 / 否
---
## 4. 关联分析
**解析 IP**: <ip>
**同 IP 域名**: X 个
**子域名发现**: X 个
---
## 5. DNS 安全配置
| 记录 | 状态 |
|------|------|
| SPF | 已配置 / 未配置 |
| DMARC | 已配置 / 未配置 |
---
## 6. 结论与处置建议
**威胁类型**: 钓鱼域名 / C2 域名 / 恶意域名
**活跃状态**: 活跃 / 不活跃
**风险等级**: 高

**评分明细**:
- **快速分析基础分**: 30
- **DGA / 同形字 / CDN / 关联资产增量分**: +15
- **DNS 安全配置增量分**: +0
- **总分**: 45

**结论来源说明**:
- 域名主结论来自 `domain-analysis`
- IP 侧补充结果来自 `ip-analysis`（如已联用）
- URL 侧补充结果来自 `url-analysis`（如已联用）

**处置建议**:
1. [x] 立即阻断 - DNS 黑名单/防火墙规则
2. [x] 内部排查 - 检查是否有解析记录
3. [x] 情报共享 - 上报威胁情报平台
---
## 7. 分析局限性
由 LLM 根据实际分析情况动态填写，例如：
- WHOIS 隐私保护，无法获取注册人信息
- 部分 DNS 记录查询超时
- 若未联用 `ip-analysis`，则 IP 侧补充结果视为未补充
- 若未联用 `url-analysis`，则 URL 侧补充结果视为未补充
---
## 8. IOC 汇总
**域名**: <domain>
**解析 IP**: <ip>
**关联域名**: <list>
**关联 URL**: <list>
```

## 评分规则摘要

### 快速分析基础分

| 指标 | 分值 |
|------|------|
| 域名年龄 < 7 天 | +30 |
| 域名年龄 7-30 天 | +20 |
| 域名年龄 30-90 天 | +10 |
| 域名年龄 90-180 天 | +5 |
| WHOIS 最终查询失败/缺失 | +10 |
| WHOIS 隐私保护 | +5 |
| 高风险 TLD | +15 |
| 多源标记恶意（≥3 源） | +40 |
| 多源标记恶意（2 源） | +30 |
| 单源标记恶意 | +20 |
| 命中 C2 标签 | +30 |
| 命中 phishing 标签 | +25 |
| 命中 malware 标签 | +25 |
| 命中 spam 标签 | +15 |
| 恶意样本关联 > 10 | +25 |
| 恶意样本关联 1-10 | +15 |

### 深度分析增量分

| 指标 | 分值 |
|------|------|
| 检测到 DGA 特征 | +35 |
| 检测到同形字攻击 | +30 |
| 命中高风险动态 DNS 服务 | +20 |
| 命中中风险动态 DNS 服务 | +10 |
| 非 CDN 且存在 Fast-Flux 特征 | +15 |
| CDN 命中 | -15 |
| 同 IP 域名 10-100 个（仅非 CDN） | +10 |
| 同 IP 域名 > 100 个（仅非 CDN） | +20 |
| 发现大量高风险子域名（≥10） | +10 |
| 未配置 SPF | +5 |
| 未配置 DMARC | +5 |
| SPF 过于宽松（+all） | +10 |

## Markdown 格式规范（必须遵守）

**段落分隔**：
- 不同主题/段落之间必须用**空行**分隔（双换行）
- 禁止把多个信息项挤在同一行
- 每个独立信息点单独成段

**键值对数据格式**（如基础信息）：
- 使用列表格式，不要用表格
- 每个列表项的完整内容必须在同一行，禁止在列表项内部换行
- 列表项之间不要空行

**表格使用场景**：
- ✅ 适合：多条记录对比（威胁情报多源、DNS记录多条、评分明细）
- ❌ 不适合：单个对象的键值对数据

**禁止使用的格式**：
- ❌ 禁止使用 `>` 引用块语法
- ❌ 禁止滥用代码块包裹普通文本
- ❌ 禁止在列表项内部换行
