# 钓鱼邮件分析报告格式规范

## 报告结构（12个章节）

| # | 章节 | 必含 | 内容要点 |
|---|------|------|----------|
| 1 | 标题 + 威胁评级 | ✅ | 等级(Critical/High/Medium/Low) + 攻击类型 |
| 2 | 邮件基本信息 | ✅ | 发件人/收件人/主题/日期/Message-ID |
| 3 | 高风险指标 | ✅ | 检测项 + 风险评分（总分/100） |
| 4 | 邮件头分析 | ✅ | SPF/DKIM/DMARC 结果 + 邮件路由链 |
| 5 | 正文分析 | ✅ | 纯文本/HTML + 规避技术检测表 |
| 6 | 载荷分析 | 条件 | 图片/附件/二维码（按实际情况） |
| 7 | IOC 提取 | ✅ | 域名/IP/邮箱/追踪ID/哈希 |
| 8 | MITRE ATT&CK 映射 | ✅ | 技术ID + 名称 + 描述 |
| 9 | 攻击链分析 | ✅ | 流程图形式展示攻击步骤 |
| 10 | 组织归因 | ✅ | 匹配组织 + 置信度 + 归因依据 |
| 11 | 分析存在的问题 | ✅ | 本次分析的局限性/未完成项 |
| 12 | 结论 | ✅ | 特征总结表 + 处置建议列表 |

## 章节详细要求

### 第1章：标题 + 威胁评级
- 必须包含威胁等级图标和分数
- 必须标明攻击类型（如：凭据钓鱼/恶意软件投递/BEC）

### 第3章：高风险指标
- 必须列出所有检测项及其分值
- 必须计算总分并给出评级

### 第7章：MITRE ATT&CK 映射
- 至少映射 2 个相关技术
- 常用技术：
  - T1566.001 - Spearphishing Attachment
  - T1566.002 - Spearphishing Link
  - T1566.003 - Spearphishing via Service
  - T1204.001 - Malicious Link
  - T1204.002 - Malicious File
  - T1598 - Phishing for Information

### 第9章：组织归因
- 必须尝试匹配已知威胁组织
- 参考 `threat-actor-attribution.md` 进行归因
- 归因维度：
  - 诱饵主题
  - 载荷格式
  - 投递渠道
  - 技术手段
  - RAT家族
- 置信度等级：高(>70%) / 中(40-70%) / 低(<40%)
- 必须说明归因局限性

### 第10章：分析存在的问题
- 必须诚实列出分析局限性
- 常见问题：
  - 链接已失效
  - 附件未动态分析
  - 缺少上下文信息
  - 无法确认攻击者身份
  - 组织归因不确定
