# 认证日志分析报告格式规范

## 报告结构（7个章节）

| # | 章节 | 必含 | 内容要点 |
|---|------|------|----------|
| 1 | 标题 + 威胁评级 | ✅ | 攻击类型 + 风险等级 |
| 2 | 日志概览 | ✅ | 时间范围/记录数/用户数/IP数 |
| 3 | 威胁检测结果 | ✅ | 凭据填充/暴力破解/不可能旅行 |
| 4 | 高风险账户 | ✅ | 受影响账户列表 |
| 5 | IOC 提取 | ✅ | 攻击 IP 列表 |
| 6 | 分析存在的问题 | ✅ | 局限性说明 |
| 7 | 结论 | ✅ | 特征总结 + 处置建议 |

---

## 评级阈值

| 等级 | 标记 | 条件 |
|------|------|------|
| Critical | 🔴 | 确认账户被盗 / 大规模凭据填充 |
| High | 🟠 | 不可能旅行 / 多账户暴力破解 |
| Medium | 🟡 | 单账户暴力破解尝试 |
| Low | 🟢 | 少量异常登录 |

---

## 报告模板

```markdown
# 🟠 认证日志分析报告 - [攻击类型]

**威胁等级**: 🟠 High
**分析时间**: YYYY-MM-DD HH:MM
**日志来源**: [邮件/VPN/SSO]

---

## 1. 日志概览

| 属性 | 值 |
|------|-----|
| 时间范围 | YYYY-MM-DD - YYYY-MM-DD |
| 总记录数 | N |
| 唯一用户数 | N |
| 唯一 IP 数 | N |
| 成功登录 | N (X%) |
| 失败登录 | N (X%) |

---

## 2. 威胁检测结果

| 低成功率 | ✅ | 成功率 2% |
| 每用户尝试次数 | ✅ | 平均 1.5 次 |

**可疑 IP**:
| IP | 尝试用户数 | 成功数 | 时间范围 |
|----|-----------|--------|----------|
| X.X.X.X | 50 | 1 | HH:MM-HH:MM |

### 不可能旅行检测
```

```
## 5. 分析存在的问题

- [ ] 缺少地理位置信息，无法完成不可能旅行检测
- [ ] 日志时间跨度不足，可能遗漏长期攻击
- [ ] 无法区分 VPN 和真实异常

---

## 6. 结论

### 特征总结

| 特征 | 描述 |
|------|------|
| 攻击类型 | 凭据填充 |
| 攻击规模 | 50 个账户被尝试 |
| 成功率 | 2% (1个账户) |
| 攻击来源 | 单一 IP (可能代理) |

### 处置建议

1. 🚫 **立即封禁** - 在防火墙/WAF 封锁攻击 IP
2. 🔐 **重置密码** - 对成功登录的账户强制重置
3. 🔒 **启用 MFA** - 对高价值账户强制多因素认证
4. 📢 **用户通知** - 通知受影响用户检查账户活动
5. 📝 **情报上报** - 将攻击 IP 提交威胁情报平台
```
