# URL 分析阶段说明

## 快速分析 vs 深度分析

| 模式 | 默认行为 | 目标 | 允许动作 |
|------|----------|------|----------|
| 快速分析 | 默认执行 | 快速完成 URL 初判并给出处置建议 | URL 解析、条件式内容获取、页面内容检测、`domain-analysis`、URL 威胁情报 |
| 深度分析 | 仅在明确触发时执行 | 补充视觉证据、归因增强、威胁扩线 | 在快速分析基础上追加截图、DNS 历史、WebSearch、扩线 |

**快速分析强约束**：
- 默认仅执行快速分析。
- 快速分析完成后，必须立即输出快速分析报告并结束。
- 快速分析中禁止继续执行页面截图、WebSearch 归因增强、DNS 历史查询、威胁扩线、同 IP / 同证书资产扩展侦察。

**深度分析触发条件**：
- 用户明确要求“深度分析”“完整分析”“全面分析”
- 用户明确要求“截图”“页面证据”
- 用户明确要求“归因增强”或“搜索安全报告”
- 用户明确要求“扩线”“同 IP 资产”“同证书站点”
- 快速分析命中高风险，且脚本输出 `deep_analysis_recommended = true`

---

## 快速分析阶段

### Phase 1: URL 解析与预处理

**目标**：完成 URL 基础解析，识别静态高风险特征。

**执行动作**：
- 解析协议、域名、端口、路径、查询参数
- 识别 IP 直连、非标准端口、短链接、高风险 TLD
- 检测同形字攻击、混合脚本域名、品牌仿冒线索
- 生成脱敏 URL（defang）

**输入**：
- 原始 URL

**输出**：
- URL 组件信息
- 静态风险因素
- 关联 IOC（域名 / IP）

---

### Phase 2: 条件式内容获取与跳转分析

> 仅在 `--fetch` 或需要结合 HTML 分析时执行。

**目标**：获取最终页面、记录跳转链并评估跳转风险。

**执行动作**：
- 发起 HTTP 获取
- 记录 HTTP 301/302/307 等跳转
- 检测 JS 跳转与 Meta Refresh
- 分析跨域跳转、长跳转链、最终落地域名

**输入**：
- 原始 URL
- `--fetch` 获取结果

**输出**：
- `fetch_info`
- `redirect_chain`
- `redirect_chain_analysis`

---

### Phase 3: 页面内容检测与规避技术检测

> 需要 HTML 内容（`--fetch` 或 `--html`）。

**目标**：识别钓鱼页面、敏感信息收集与规避技术。

**执行动作**：
- 钓鱼内容检测
- 敏感字段识别
- 诈骗关键词检测
- 品牌仿冒检测
- DOM 噪音注入检测
- CSS 内容隐藏检测
- 初步黑产组织归因

**输入**：
- 页面 HTML
- 页面标题（如可用）

**输出**：
- `phishing_detection`
- `evasion_analysis`
- `attribution`
- 初步风险增量

---

### Phase 4: 域名权威信息与 URL 威胁情报补充

**目标**：整合外部权威结果，避免 URL 技能越权替代域名分析。

**执行动作**：
- 对非 IP 直连 URL，调用 `domain-analysis` 获取域名年龄 / WHOIS / DNS 权威信息
- 调用 URL 威胁情报查询

**调用规则**：

```bash
python <DOMAIN_ANALYSIS_SKILL_DIR>/scripts/domain_analyze.py "<域名>"
```

**重点整合字段**：
- 注册商
- 注册日期
- 域名年龄
- WHOIS 隐私保护
- DNS 记录
- 域名侧风险评分
- URL 威胁标签 / 检测率 / 最近活动时间

**输出**：
- `domain_whois`（动作提示或跳过说明）
- `threat_intel`（待查询提示或查询结果整合）

---

### Phase 5: 快速风险评估与快速报告输出

**目标**：在最小闭环内给出风险结论，不自动滑入深度分析。

**执行动作**：
- 计算总风险分数与风险等级
- 生成处置建议
- 生成深度分析建议信号
- 输出快速分析报告

**必须输出的建议信号**：
- `deep_analysis_recommended`
- `deep_analysis_reasons`
- `websearch_suggestions`（仅作为深度分析建议）
- `expansion_suggestions`（仅作为深度分析建议）

**输出要求**：
- 只输出快速分析报告
- 如果建议升级，必须明确写出升级原因
- 不得在快速模式中直接执行截图 / WebSearch / 扩线

---

## 深度分析追加阶段

### Phase 6: 页面截图

**触发条件**：
- 用户明确要求截图 / 页面证据
- 快速分析高风险，需补充视觉证据

**执行动作**：
- 调用 `webcap_tool` 获取页面截图
- 在报告中使用 Markdown 图片语法嵌入返回链接

**写法要求**：

```markdown
![页面截图](截图URL)
```

---

### Phase 7: DNS 历史查询

**触发条件**：
- 用户明确要求深度分析
- 需要确认历史解析关系或基础设施变化

**执行动作**：
- 调用 `cybersec_cloud_mcp_dns_history`
- 查询域名或关联 IP 的 DNS 历史

**注意**：
- DNS 历史属于历史情报，不代表当前状态
- 只能作为辅助证据，不能单独替代当前页面分析

---

### Phase 8: WebSearch 归因增强

**触发条件**：
- `websearch_suggestions.enabled = true`
- 用户明确要求归因增强 / 搜索安全报告

**执行动作**：
- 根据 `websearch_suggestions` 执行 WebSearch
- 提取安全厂商报告、归因信息、关联 IOC
- 更新归因结论与置信度

**要求**：
- `websearch_suggestions` 是建议，不是默认必做动作
- 只有进入深度分析后才允许执行

---

### Phase 9: 威胁扩线

**触发条件**：
- `expansion_suggestions.enabled = true`
- 用户明确要求扩线 / 同 IP / 同证书 / 相似标题分析

**执行动作**：
- 根据 `expansion_suggestions` 调用 `cybersec_cloud_mcp_cyberspace-search`
- 按维度汇总高风险关联资产
- 输出扩线 IOC 与过滤统计

**要求**：
- `expansion_suggestions` 仅表示“适合在深度分析中追加”
- 快速分析阶段不得默认自动执行

---

### Phase 10: 深度风险评估与深度报告输出

**目标**：整合快速分析、视觉证据、归因增强与扩线结果，输出深度报告。

**执行动作**：
- 在快速分析结论基础上追加截图、DNS 历史、归因增强、扩线结果
- 更新最终风险评估
- 输出深度分析报告

**输出要求**：
- 深度报告必须明确哪些检查是本次追加执行的
- 未执行的深度检查必须写“未执行”，不要伪装为阴性结果

---

## 模式边界速查

| 能力 | 快速分析 | 深度分析 |
|------|----------|----------|
| URL 解析 | 必做 | 复用快速结果 |
| `--fetch` 内容获取 | 条件执行 | 可复用快速结果 |
| 跳转分析 | 条件执行 | 可复用快速结果 |
| 钓鱼内容检测 | 有 HTML 时执行 | 可复用快速结果 |
| 规避技术检测 | 有 HTML 时执行 | 可复用快速结果 |
| `domain-analysis` | 非 IP URL 必做 | 复用快速结果 |
| URL 威胁情报 | 必做 | 复用快速结果 |
| 页面截图 | 禁止 | 按需追加 |
| DNS 历史 | 禁止 | 按需追加 |
| WebSearch 归因增强 | 禁止 | 按需追加 |
| 威胁扩线 | 禁止 | 按需追加 |
